#  Peak Performance Gym Database Management System

A comprehensive relational database management system developed for **CMPG311** that streamlines the daily operations of a fitness gym. The system manages members, memberships, trainers, classes, bookings, attendance, payments, equipment usage, and reporting using **Oracle SQL Developer**.



## 📖 Project Overview

The Peak Performance Gym Database was designed to replace manual record keeping and spreadsheets with a centralized database system.

The database provides a structured solution for managing:

- Member registration
- Membership plans
- Personal trainers
- Gym classes
- Bookings
- Payments
- Attendance tracking
- Equipment usage
- Business reporting

The project was developed as part of the **CMPG311 Database Systems** module. :contentReference[oaicite:1]{index=1}

## 🗄 Database Design

The system consists of the following main tables:

- MEMBER
- MEMBERSHIP
- MEMBERSHIP_TYPE
- MEMBERSHIP_STATUS
- EMPLOYEE
- TRAINER
- EXPERTISE
- TRAINER_EXPERTISE
- GYM_CLASS
- BOOKING
- PAYMENT
- ATTENDANCE
- EQUIPMENT
- USAGE_RECORD
- BENEFIT
- MEMBERSHIP_TYPE_BENEFIT

The design uses:

- Primary Keys
- Foreign Keys
- Check Constraints
- Unique Constraints
- Sequences
- Triggers
- Views
- Stored Procedures



## 📊 Database Features

### Constraints

- Data validation using CHECK constraints
- Foreign key relationships
- Unique email and phone numbers
- Membership date validation
- Booking status validation
- Payment validation

### Automation

The project includes several Oracle triggers to automatically generate IDs using sequences.

Examples include:

- Membership Trigger
- Booking Trigger
- Payment Trigger
- Attendance Trigger
- Usage Record Trigger
